# Contributing to ETIL TUI

Thank you for your interest in contributing to the ETIL MCP Client TUI.

## Getting Started

1. Fork the repository
2. Clone your fork and create a feature branch
3. Set up the development environment:
   ```bash
   ./setup.sh          # creates venv + installs deps
   ./run.sh --help     # verify it works
   ```

## Development

- **Python 3.12+** required
- **Textual** >= 0.85.0 for the TUI framework
- Run tests: `python -m pytest tests/`
- Follow existing code style (PEP 8, type hints where practical)

## Pull Requests

- One feature or fix per PR
- Include a clear description of what changed and why
- Add tests for new functionality
- Ensure existing tests pass

## Reporting Issues

Use the [GitHub issue tracker](https://github.com/krystalmonolith/etil-tui/issues).
Include your Python version, OS, and steps to reproduce.
